    Программа состоит из скрипта для фонового сохрания данных о погоде. И веб-приложения, которое позволяет
посмотреть на результат работы скрипта. Источник данных REST API NGS.RU.

    Диспетчер истории должен запускаться в единственном экземпляре, иначе добавит вчерашнюю дату N раз,
где N - число экземпляров скрипта.

    Для Perl необходимо скачать и установить LWP (достаточно того, который по http) и JSON,
так как они не идут из коробки обычно.

    Для удобства тестирования, чтобы не ждать сутки перед добавлением истории: можно поменять первый оператор "ne" на "eq"
в 305-ой строке скрипта perl. Даты будут добавляться сегодняшние, минимум и максимум по понятным причинам будут равны
и отобразятся одним числом (это будет текущая температура).

    Перед использованием Веб-приложения необходимо запустить Диспетчер хотя бы один раз, так как приложение
берет из базы названия городов, так же, при запуске скрипта, происходит добавление новых городов, если
они недавно добавились в REST API.

    Использование приложения:
Нажать на + в кружке, чтобы добавить город (хоть все доступные).
Нажать на = у города, чтобы изменить или удалить его.
Нажать на название города, чтобы перейти на страницу информации по городу.
Нажать на << чтобы вернуться.

    Можно вводить как название города, так и его алиас. Значение алиаса подставляется через
xmlhttprequest в блокирующем режиме. Скрипт, который формирует ответ - getalias.php
